# Решение «Отклик на кредитный оффер»

Градиентный бустинг как правило на табличных работает лучше нейросетей и быстрее в реализации. Параметры подбирал Optuna

- **Пропуски не заполняем, а кодируем.** Пропуск информативен (напр. `overdraft_app_term_max_360`: есть значение → отклик 31%, нет → 5%). Числовые `NaN` оставляем (бустинг обрабатывает нативно) + индикаторы `*_isna`, `n_missing`, `n_present_rare`.
- **Признаки:** разности-взаимодействия (`loan_amount_vs_overdraft_limit` — одиночный AUC 0.71, спред ставки, дельты окон 30/90). Категории: `NaN="missing"`.
- **Модель — ансамбль XGBoost + CatBoost (ранговое усреднение).** Бустинг по деревьям: нативно работает с `NaN`, устойчив к шкалам/выбросам. Две разные модели → стабильный прирост (holdout: XGB 0.7516, CatBoost 0.7533, **ансамбль 0.7556**).
- **Валидация по времени**. Случайный K-Fold завышает оценку.
